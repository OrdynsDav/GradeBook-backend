# Документация API для админ-панели GradeBook

Для админ-панели используются только эндпоинты, доступные с ролью **admin**. Все запросы выполняются с заголовком `Authorization: Bearer <accessToken>`, где `accessToken` получен после входа под учёткой администратора.

**Базовый URL:** `https://gradebook-backend-xhw2.onrender.com/api/v1/`

---

## 1. Вход под администратором

Админ входит так же, как остальные пользователи.

**POST** `/auth/login`

Тело (JSON):

```json
{
  "login": "admin",
  "password": "Password123!"
}
```

Ответ `200`: `{ "accessToken", "refreshToken", "expiresIn", "user" }`. В `user` приходят поля: `id`, `login`, `role`, `firstName`, `lastName`, `middleName`, `groupId` (у студента — UUID группы), `createdAt`, `updatedAt`. Дальше все запросы админ-панели отправляйте с этим `accessToken` в заголовке `Authorization: Bearer <accessToken>`.

## 2. Пользователи (admin)

### 2.1 Список всех пользователей

**GET** `/users`

Доступно только **admin**. Возвращает массив всех пользователей в том же формате, что и `GET /users/me` (id, login, role, firstName, lastName, middleName, group, createdAt, updatedAt). Пароли не отдаются.

Ответ `200`: массив объектов пользователей. Сортировка: по роли, затем по фамилии и имени.

Ошибки: `403` — не admin.

### 2.2 Создание пользователя

**POST** `/users`

Создаёт нового пользователя с ролью **student** или **teacher**. Ученику автоматически подбирается группа по паре (курс + подгруппа); если такой группы нет — она создаётся.

### Тело запроса (JSON)

| Поле        | Тип     | Обязательное | Описание |
|------------|---------|--------------|----------|
| `role`     | string  | да           | `"student"` или `"teacher"` |
| `firstName`| string  | да           | Имя, 1–64 символа |
| `lastName` | string  | да           | Фамилия, 1–64 символа |
| `middleName`| string | нет          | Отчество, до 64 символов |
| `course`   | number  | для student  | Курс 1–4 |
| `group`    | string  | для student  | Группа (буква/название), 1–32 символа |
| `subjects` | array   | для teacher  | Массив предметов. Элемент: по id — `groupId` (одна) или `groupIds` (несколько); по названию — `groups: ["И14-1", "И14-2"]`. Пример: `{ "name": "Математика", "groups": ["И14-1", "И14-2"] }`. Регистр имён групп не важен. |
| `login`    | string  | да           | Уникальный логин, 3–64 символа, только `a-zA-Z0-9._-` |
| `password` | string  | да           | Пароль, 8–128 символов |

Пример для ученика:

```json
{
  "role": "student",
  "firstName": "Никита",
  "lastName": "Иванов",
  "middleName": "Петрович",
  "course": 1,
  "group": "A",
  "login": "student.new",
  "password": "SecurePass123!"
}
```

Пример для учителя (без предметов):

```json
{
  "role": "teacher",
  "firstName": "Мария",
  "lastName": "Сидорова",
  "login": "teacher.english",
  "password": "SecurePass123!"
}
```

Пример для учителя с несколькими предметами:

```json
{
  "role": "teacher",
  "firstName": "Иван",
  "lastName": "Петров",
  "middleName": "Сергеевич",
  "login": "teacher.math",
  "password": "SecurePass123!",
  "subjects": [
    { "name": "Mathematics", "groupId": "e2f08ca8-6866-4df6-8d43-1c2f4d8f4488" },
    { "name": "Математика", "groups": ["И14-1", "И14-2"] }
  ]
}
```

Один предмет для многих групп: укажите `groupIds` (массив UUID) или **`groups`** (массив названий групп, например `["И14-1", "И14-2"]`). Группы по названию ищутся без учёта регистра.

Ответ `201`: объект пользователя (как в `GET /users/me`), с полем `group` для ученика.

Ошибки: `400` — невалидные данные (в т.ч. для teacher: несуществующий `groupId`/`groupIds` в `subjects` или предмет с таким именем уже есть в группе), `409` — логин уже занят или пользователь с таким ФИО уже есть, `403` — не admin.

## 3. Группы (список и создание)

Группы нужны для выбора при создании учителя (предметы по группам) и студентов. Группу можно создать явно через API или она создаётся автоматически при добавлении студента с новой парой (курс + группа).

### 3.1 Список групп

**GET** `/groups`

Доступно всем авторизованным. Возвращает все группы для подстановки в выпадающие списки (создание учителя, уроков и т.д.).

Ответ `200`: массив объектов вида:

```json
[
  {
    "id": "uuid",
    "name": "A",
    "course": 1,
    "groupName": "A",
    "curatorId": null,
    "createdAt": "2026-02-21T16:00:00.000Z",
    "updatedAt": "2026-02-21T16:00:00.000Z"
  }
]
```

### 3.2 Создать группу (только admin)

**POST** `/groups`

Тело (JSON):

| Поле        | Тип    | Обязательное | Описание |
|------------|--------|--------------|----------|
| `course`   | number | да           | Курс 1–4 |
| `groupName`| string | да           | Буква/название группы, 1–32 символа (будет приведено к верхнему регистру) |

Пример:

```json
{
  "course": 1,
  "groupName": "A"
}
```

Ответ `201`: созданная группа (id, name, course, groupName, curatorId, createdAt, updatedAt). Поле `name` совпадает с `groupName` (без префикса курса), например `И14-1`.

Ошибки: `400` — невалидные данные или группа с такой парой (курс + groupName) уже есть, `403` — не admin.

## 4. Расписание (только admin — создание/изменение/удаление)

Предметы и группы в системе уже заданы; урок привязан к предмету. Учитель и группа берутся из предмета автоматически.

### 4.1 Список предметов (для выбора при создании урока)

**GET** `/subjects`

Для админа доступны все предметы. Опциональные query-параметры:

- `groupId` — фильтр по группе (UUID)
- `teacherId` — фильтр по учителю (UUID)

Ответ `200`: массив объектов вида:

```json
[
  {
    "id": "uuid",
    "name": "Mathematics",
    "groupId": "uuid",
    "teacherId": "uuid",
    "group": { "id": "uuid", "name": "A" },
    "teacher": {
      "id": "uuid",
      "firstName": "Ivan",
      "lastName": "Petrov",
      "middleName": "Sergeevich"
    },
    "createdAt": "2026-02-21T16:00:00.000Z",
    "updatedAt": "2026-02-21T16:00:00.000Z"
  }
]
```

В админ-панели этот список используется, чтобы выбрать `subjectId` при создании/редактировании урока.

### 4.2 Расписание на неделю / день (для отображения и фильтров)

**GET** `/schedule/week?date=YYYY-MM-DD`

**GET** `/schedule/day?date=YYYY-MM-DD`

Для админа можно дополнительно передать:

- `groupId` — фильтр по группе
- `teacherId` — фильтр по учителю

Ответ `200`: массив уроков (структура ниже в п. 4.3).

### 4.3 Создать урок

**POST** `/schedule`

Тело (JSON):

| Поле       | Тип    | Обязательное | Описание |
|------------|--------|--------------|----------|
| `subjectId`| string | да           | UUID предмета (из GET /subjects) |
| `startsAt` | string | да           | Начало урока, ISO 8601 (UTC), напр. `2026-02-21T08:00:00.000Z` |
| `endsAt`   | string | да           | Конец урока, ISO 8601 (UTC) |
| `room`     | string | нет          | Аудитория, до 64 символов (в ответе может быть `null`) |

Учитель и группа подставляются из предмета; в теле их указывать не нужно.

Пример:

```json
{
  "subjectId": "d12de61b-4f43-4047-9d96-c4e94b9be740",
  "startsAt": "2026-02-21T08:00:00.000Z",
  "endsAt": "2026-02-21T08:45:00.000Z",
  "room": "101"
}
```

Ответ `201`: созданный урок в формате:

```json
{
  "id": "uuid",
  "startsAt": "2026-02-21T08:00:00.000Z",
  "endsAt": "2026-02-21T08:45:00.000Z",
  "room": "101",
  "subject": { "id": "uuid", "name": "Mathematics" },
  "group": { "id": "uuid", "name": "A", "course": 1, "groupName": "A" },
  "teacher": { "id": "uuid", "firstName": "Ivan", "lastName": "Petrov", "middleName": "Sergeevich" }
}
```

Ошибки: `400` — невалидные данные/даты, `404` — предмет не найден, `403` — не admin.

### 4.4 Обновить урок

**PATCH** `/schedule/:id`

Тело (JSON), все поля опциональны:

| Поле       | Тип    | Описание |
|------------|--------|----------|
| `subjectId`| string | UUID другого предмета (меняет учителя и группу) |
| `startsAt` | string | ISO 8601 |
| `endsAt`   | string | ISO 8601 |
| `room`     | string | Аудитория |

Пример: `{ "room": "202" }` — только смена кабинета.

Ответ `200`: обновлённый урок (та же структура, что при создании). Ошибки: `404` — урок/предмет не найден, `403` — не admin.

### 4.5 Удалить урок

**DELETE** `/schedule/:id`

Ответ `200`: `{ "success": true }`. Ошибки: `404` — урок не найден, `403` — не admin.

## 5. Типы для админ-панели (TypeScript)

Используйте типы из `frontend-types.ts` в этой же папке:

| Метод | Путь | Описание |

## 7. Ошибки

- **401** — нет или неверный/истёкший access token. Нужно снова вызвать `/auth/login` или обновить токен через `/auth/refresh`.
- **403** — доступ запрещён (не admin).
- **404** — урок, предмет или пользователь не найден.
- **409** — при создании пользователя: логин уже занят или пользователь с таким именем, фамилией и отчеством уже есть.

Формат тела ошибки от API: `{ "statusCode", "message", "error" }` (и при необходимости `path`, `timestamp`, `requestId`).
